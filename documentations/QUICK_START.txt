📋 PROJECT OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

YouTube Sync Player is a sophisticated real-time streaming application that
enables multiple users to watch YouTube videos together with sub-millisecond
precision synchronization across the internet.

Key Technology: NTP-based clock synchronization + PI-controller playback adjustment

🔄 REAL-TIME SYNC ALGORITHM
═══════════════════════════════════════════════════════════════════════════════

PHASE 1: NTP CLOCK CALIBRATION (Upon Connection)
┌─────────────────────────────────────────────────────────────────┐
│ Client performs 5 ping-pong exchanges with server:              │
│                                                                 │
│ Exchange 1: Send ClientTime T1                                 │
│             Receive: ServerReceiveTime T2, ServerSendTime T3   │
│             Measure: ClientReceiveTime T4                      │
│                                                                 │
│ Calculate: RTT = (T4 - T1) - (T3 - T2)                        │
│           Offset = ((T2 - T1) + (T3 - T4)) / 2                │
│                                                                 │
│ Result: Client knows server time within ±3-5ms accuracy       │
└─────────────────────────────────────────────────────────────────┘

PHASE 2: CONTINUOUS PLAYBACK SYNC (Every 100ms)
┌─────────────────────────────────────────────────────────────────┐
│ For each joiner client:                                         │
│                                                                 │
│ 1. Calculate Expected Position:                                │
│    ExpectedPos = VideoProgress + (ServerTime - LastUpdateTime) │
│                                                                 │
│ 2. Measure Actual Position:                                    │
│    ActualPos = Current YouTube Player Position                 │
│                                                                 │
│ 3. Calculate Drift:                                            │
│    Drift = ExpectedPos - ActualPos                             │
│                                                                 │
│ 4. Apply Correction:                                           │
│    ├─ If |Drift| < 50ms:    Speed = 1.0x (normal)            │
│    ├─ If 50ms ≤ |Drift| < 1.5s:                              │
│    │  ├─ If Drift > 0 (behind): Speed = 1.05x (catch up)     │
│    │  └─ If Drift < 0 (ahead):  Speed = 0.95x (slow down)    │
│    └─ If |Drift| ≥ 1.5s:    Hard seek to ExpectedPos         │
│                                                                 │
│ Result: Sub-100ms drift WITHOUT jarring jumps or audio pops    │
└─────────────────────────────────────────────────────────────────┘

🔌 API ENDPOINTS & WEBSOCKET EVENTS
═══════════════════════════════════════════════════════════════════════════════

REST API:
  POST /rooms/create          → Create a new streaming room
  GET  /rooms/{roomId}/info   → Get room information
  GET  /health                → Server health check

WebSocket Events:
  Client → Server:
    • clock-sync:ping                    (NTP calibration)
    • room:join                          (Join a room)
    • room:update-video                  (Host: Change video)
    • room:playback-control              (Host: Play/Pause)
    • room:seek                          (Host: Seek position)
  
  Server → Client:
    • clock-sync:pong                    (NTP response)
    • room:state                         (Room state update)
    • room:participant-count             (Viewer count update)

📞 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: Videos not syncing?
  → Check network latency (RTT should be < 100ms)
  → Ensure both clients are connected to same server
  → Wait 5-10 seconds for clock calibration

Problem: Cannot connect to server?
  → Verify backend running: curl http://localhost:4000/health
  → Check NEXT_PUBLIC_SERVER_URL in client/.env.local
  → Ensure port 4000 is not blocked

Problem: YouTube video won't load?
  → Verify URL format: youtube.com/watch?v=VIDEO_ID
  → Check if video allows embedding
  → Look for CORS errors in console
